# Codewars Solutions

![Codewars Badge](https://www.codewars.com/users/flrntvl/badges/large)

## 📘 Overview

This repository contains my personal solutions to various **Codewars kata**.

The goal of this repository is primarily **educational**. It serves as a place to store my approaches, track my progress, and document different ways to solve algorithmic problems.

It is organized as a personal practice repository rather than a production application: each kata lives in its own folder, with a solution file and its associated tests.

## 🗂️ Project Structure

The repository is grouped by language first, then by kata difficulty, and finally by kata name.

codewars-solutions/

└── ruby/

## 🧩 Kata Layout

Each kata folder follows a very small and consistent structure:

- one file for the solution
- one file for the tests

Example:

```text
<language>/<kyu>/<kata-name>/
├── solution file
└── test file
```

File names may vary depending on the language, but the goal stays the same: keep each kata self-contained and easy to navigate.

Language-specific setup and usage notes are documented in each language folder, for example in [ruby/README.md](ruby/README.md).

## ⚠️ Important Notice

If you are currently solving challenges on Codewars, **please do not use this repository to complete kata without attempting them yourself first.**

Solving problems independently is the best way to learn and improve your problem-solving skills.

These solutions are shared:

* 📚 for **learning purposes**
* 🔍 to **review different approaches**
* 🧠 to **understand alternative implementations**
* 🆘 as a **last resort** if you are truly stuck

Please try to solve the kata on your own before reading any solution.

## 🧾 Disclaimer

These solutions represent **my personal learning process** and may not always be the most optimal or idiomatic approach.

Codewars often has many creative solutions — exploring different implementations is encouraged.
